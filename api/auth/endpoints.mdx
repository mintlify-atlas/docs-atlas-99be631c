All authentication endpoints are handled by Better-Auth through a catch-all route at `/api/auth/*`.

## Implementation

From `src/app/api/auth/[...all]/route.ts:1-5`:

```typescript
import { auth } from '@/lib/auth'
import { toNextJsHandler } from 'better-auth/next-js'

export const { GET, POST } = toNextJsHandler(auth)
```

This configuration automatically handles all Better-Auth endpoints.

## Base URL

All endpoints are prefixed with `/api/auth`. The base URL is configured in `src/lib/auth/auth.ts:20`:

```typescript
baseURL: getBaseUrl() // Automatically uses NEXT_PUBLIC_APP_URL
```

## Email & Password Endpoints

### Sign Up

<CodeGroup>
```typescript Client
import { client } from '@/lib/auth'

const result = await client.signUp.email({
  email: 'user@example.com',
  password: 'securePassword123',
  name: 'John Doe'
})
```

```bash cURL
curl -X POST https://yourapp.com/api/auth/sign-up/email \
  -H "Content-Type: application/json" \
  -d '{
    "email": "user@example.com",
    "password": "securePassword123",
    "name": "John Doe"
  }'
```
</CodeGroup>

<ParamField body="email" type="string" required>
  User's email address
</ParamField>

<ParamField body="password" type="string" required>
  User's password (minimum requirements defined by Better-Auth)
</ParamField>

</ParamField>

    </ResponseField>
    <ResponseField name="emailVerified" type="boolean">
      Email verification status
    </ResponseField>

    </ResponseField>
    <ResponseField name="createdAt" type="string">
      Account creation timestamp

</ResponseField>

## OAuth Endpoints

### Initiate OAuth Flow

<CodeGroup>
```typescript Client
import { client } from '@/lib/auth'

// Redirects to OAuth provider
await client.signIn.social({
  provider: 'google',
  callbackURL: '/dashboard'
})
```

```bash Direct URL
# Navigate to:
GET /api/auth/signin/google?callbackURL=/dashboard
```
</CodeGroup>

<ParamField query="provider" type="string" required>
  OAuth provider: `google`, `github`, `microsoft`, or `facebook`
</ParamField>

<ParamField query="callbackURL" type="string">
  URL to redirect after authentication
</ParamField>

**Supported Providers** (from `src/lib/auth/auth.ts:45-78`):

<Accordion title="Google OAuth">
  **Required Environment Variables:**
  - `GOOGLE_CLIENT_ID`
  - `GOOGLE_CLIENT_SECRET`
  
  **Scopes:** `email`, `profile`
</Accordion>

<Accordion title="GitHub OAuth">
  **Required Environment Variables:**
  - `GITHUB_CLIENT_ID`
  - `GITHUB_CLIENT_SECRET`
  
  **Scopes:** `user:email`
</Accordion>

<Accordion title="Microsoft OAuth">
  **Required Environment Variables:**
  - `MICROSOFT_CLIENT_ID`
  - `MICROSOFT_CLIENT_SECRET`
  - `MICROSOFT_TENANT_ID` (optional, defaults to 'common')
</Accordion>

<Accordion title="Facebook OAuth">
  **Required Environment Variables:**
  - `FACEBOOK_CLIENT_ID`
  - `FACEBOOK_CLIENT_SECRET`
  
  **Scopes:** `email`, `public_profile`
</Accordion>

### OAuth Callback

Better-Auth automatically handles OAuth callbacks at:

```
GET /api/auth/callback/{provider}
```

No manual implementation required.

```typescript Client

function MyComponent() {
  const { data: session, isPending } = useSession()
```

```typescript Server
import { auth } from '@/lib/auth'
```

## Error Responses

All endpoints return standard HTTP status codes:

<ResponseField name="400" type="Bad Request">
  Invalid request parameters or validation error
</ResponseField>

<ResponseField name="401" type="Unauthorized">
  Authentication required or invalid credentials
</ResponseField>

<ResponseField name="403" type="Forbidden">
  Insufficient permissions
</ResponseField>

<ResponseField name="404" type="Not Found">
  Resource not found
</ResponseField>

<ResponseField name="500" type="Internal Server Error">
  Server error
</ResponseField>

**Error Response Format:**

```json
{
  "error": {
    "message": "Error description",
    "code": "ERROR_CODE"
  }
}
```
