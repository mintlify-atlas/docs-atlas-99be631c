ShipFree uses environment variables for configuration. All variables are validated at build time using `@t3-oss/env-nextjs` and Zod for type safety.

## Setup

1. Copy `.env.example` to `.env` in your project root:
```bash
cp .env.example .env
```

2. Fill in the required values for your environment
3. Variables are validated in `src/config/env.ts`

## Validation

All environment variables are validated using Zod schemas:

```typescript
import { createEnv } from '@t3-oss/env-nextjs'
import { z } from 'zod'

export const env = createEnv({
  server: { /* server-only variables */ },
  client: { /* client-accessible variables */ },
  shared: { /* available on both */ },
  runtimeEnv: { /* process.env mappings */ }
})
```

Variables in the `server` object are only accessible server-side. Variables in `client` must be prefixed with `NEXT_PUBLIC_` and are exposed to the browser.

## Core Configuration

### Database

<ParamField path="DATABASE_URL" type="string" required>
  PostgreSQL connection string used by Drizzle ORM.
  
  ```bash
  DATABASE_URL=postgres://postgres:postgres@localhost:5432/shipfree
  ```
</ParamField>

<ParamField path="PROD_DATABASE_URL" type="string">
  Production database URL used when running migrations in production.
  
  ```bash
  PROD_DATABASE_URL=postgres://user:password@host:5432/database
  ```
</ParamField>

### Application URL

<ParamField path="NEXT_PUBLIC_APP_URL" type="string" default="http://localhost:3000">
  Public URL of your application. Used for OAuth callbacks, email links, and API endpoints.
  
  **Local development:**
  ```bash
  NEXT_PUBLIC_APP_URL=http://localhost:3000
  ```
  
  **Production:**
  ```bash
  NEXT_PUBLIC_APP_URL=https://yourdomain.com
  ```
</ParamField>

### Node Environment

<ParamField path="NODE_ENV" type="enum">
  Runtime environment. Values: `development`, `production`, `test`
  
  Validated as:
  ```typescript
  z.enum(['development', 'test', 'production']).optional()
  ```
</ParamField>

<ParamField path="NEXT_PUBLIC_ENV" type="string" default="development">
  Client-side environment indicator exposed to the browser.
</ParamField>

## Authentication (Better-Auth)

<ParamField path="BETTER_AUTH_SECRET" type="string" required default="dev-secret-change-in-production">
  Secret key used to encrypt sessions and sign tokens.
  
  **Generate a secure secret:**
  ```bash
  openssl rand -base64 32
  ```
  
  **Never use the default in production!**
</ParamField>

<ParamField path="BETTER_AUTH_URL" type="string" default="http://localhost:3000">
  Base URL of your application. Must match `NEXT_PUBLIC_APP_URL`.
</ParamField>

<ParamField path="BILLING_ENABLED" type="boolean" default="false">
  Enable billing enforcement across the application.
  
  Validated as:
  ```typescript
  z.boolean().default(false)
  ```
</ParamField>

<ParamField path="EMAIL_VERIFICATION_ENABLED" type="boolean" default="false">
  Require email verification for new user accounts.
</ParamField>

## Type Safety

Access environment variables through the validated `env` object:

```typescript
import { env } from '@/config/env'

// ✅ Type-safe and validated
const dbUrl = env.DATABASE_URL
const appUrl = env.NEXT_PUBLIC_APP_URL

// ❌ Don't use process.env directly
const dbUrl = process.env.DATABASE_URL // Not type-safe!
```

The `env` object ensures:
- All required variables are present at build time
- Values match expected types (string, number, boolean, enum)
- Default values are applied
- Client variables are properly prefixed
